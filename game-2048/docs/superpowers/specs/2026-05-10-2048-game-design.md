# 2048 Game Design Spec

## Overview
A classic 2048 number-merging puzzle game as a single HTML file. Double-click to play, zero dependencies, runs in any browser.

## File Structure
```
game-2048/
  index.html    # Self-contained game (HTML + CSS + JS)
  README.md     # Screenshot + how to play
```

## UI Layout
- **Header row:** Title "2048", score panel, best score, "New Game" button
- **Game board:** 4×4 grid, dark background (#bbada0), rounded corners
- **Tiles:** Colored by value with CSS transitions for slide/merge animations

## Core Logic
- `board[4][4]` 2D array, initialized with 2 random tiles (value 2 or 4, 90%/10% chance)
- Arrow key input triggers: slide → merge → slide → spawn
- Merge: adjacent same-value tiles combine, score increments by merged value
- Win: any tile reaches 2048, continue playing allowed
- Lose: no empty cells AND no adjacent same values
- `requestAnimationFrame` used for smooth animation

## Error Handling
- Keyboard input ignored during ongoing animation (500ms debounce)
- Invalid moves (no tiles moved) do nothing, do NOT spawn new tiles

## Testing
- Manual testing: open in browser, play through to win/lose states
- Edge cases: rapid key presses, all 4 directions when board is full, undo behavior
